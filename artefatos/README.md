# 📦 Artefatos do Projeto – Qualidade de Software

Esta pasta reúne todos os artefatos produzidos durante o desenvolvimento da atividade de Testes Unitários e TDD no sistema LocalEats.

O objetivo é centralizar a documentação e evidências que comprovam a aplicação das práticas de Qualidade de Software.

---

## 📁 Estrutura

### 📊 diagramas/
Contém os diagramas que representam a estrutura e comportamento do sistema.

**Inclui:**
- Diagrama de Casos de Uso
- Diagrama de Classes
- Diagrama de Sequência

**Objetivo:**
Facilitar o entendimento do funcionamento do sistema e da lógica implementada.

---

### 📸 evidencias/
Contém provas da execução dos testes e da aplicação do TDD.

**Inclui:**
- Logs de execução do pytest
- Resultados dos testes
- Simulação do ciclo TDD (Red → Green → Refactor)
- Cobertura de testes (coverage)

**Objetivo:**
Comprovar que o sistema foi testado e validado corretamente.

---

### 📄 relatorios/
Contém a documentação final da atividade.

**Inclui:**
- Relatório completo do projeto
- Descrição da funcionalidade implementada
- Explicação dos testes unitários
- Aplicação do TDD
- Reflexão crítica

**Objetivo:**
Apresentar de forma clara todo o processo de desenvolvimento e validação.

---

## 🎯 Objetivo Geral dos Artefatos

- Documentar o sistema
- Demonstrar a aplicação de testes automatizados
- Evidenciar o uso do TDD
- Garantir rastreabilidade do desenvolvimento
- Comprovar qualidade do software

---

## 🧪 Práticas Aplicadas

- Testes Unitários Automatizados
- Test-Driven Development (TDD)
- Refatoração de código
- Validação de regras de negócio
- Organização modular

---

## 🟢 Conclusão

A pasta de artefatos demonstra que o projeto segue boas práticas de Qualidade de Software, garantindo:

- ✔ Código confiável  
- ✔ Facilidade de manutenção  
- ✔ Clareza na documentação  
- ✔ Validação contínua
